## theory 结构

沿用 Q&A 模块既有板块（H1 + 引入与类比 + 核心概念图解 + 深入讲解 + 常见误区 + 一句话总结），预计 2500–3500 字符（现有 Q&A 章节约 1000–1300，本章主题更宽，适当加长）。

1. **开头段**：先给结论——没有「最好的语言」，只有「最合适的场景」。Rust 的核心卖点：无 GC 的内存安全 + C/C++ 级性能 + 现代工具链（Cargo）。
2. **## 🧭 引入与类比**：交通工具类比——C/C++ 手动挡赛车、Java 大巴、Go 够用的小车、JS 电动自行车、Python 自动挡家用车、Rust 带全套安全系统的性能车。
3. **## 💡 核心概念图解**：两张 SVG 矢量插图（见「插图设计」），风格对齐 `images/module4-*.svg`（720 宽、米白底、白卡片、system-ui 字体）。
4. **## 🔧 深入讲解**，每语言一节（### 小节），既讲 Rust 优势也讲对方仍强的场景：
   - **vs C/C++**：内存安全不靠程序员自觉——悬垂指针、越界、double free；业界公开报告（Microsoft、Chromium 项目）称约七成高危安全漏洞与内存安全相关。Rust 用所有权 + 借用检查在编译期消灭此类问题；性能与 C/C++ 同一量级（零成本抽象、无运行时）。C++ 仍强在存量生态与成熟库。
   - **vs Java**：无 GC、无虚拟机——可预测延迟（无 STW 停顿）、低内存占用、编译为原生单文件。Java 仍强在企业级生态、人才池与成熟的运行时诊断工具。
   - **vs Go**：零成本抽象与显式错误处理——Go 的 GC 延迟低但非零开销，`if err != nil` 冗长，表达能力刻意克制；Rust 的泛型/trait 零成本，`Result` + `?` 显式传播错误。Go 仍强在编译快、上手快、并发代码直白。
   - **vs JavaScript**：类型安全与性能天花板——JS 动态类型导致大量运行时错误，单线程事件循环不擅 CPU 密集任务；Rust 编译期类型检查 + 真并行，且可编译到 WebAssembly 与 JS 互补（如 Figma、Ruffle 等场景），并非替代关系。
   - **vs Python**：性能差一到两个数量级 + GIL 限制并行；Python 在 AI/数据科学生态无可替代。Rust 常以高性能扩展身份融入 Python 生态：PyO3、pydantic-core、ruff、uv、polars。
   - **Rust 当前的不足**：学习曲线陡峭（所有权/生命周期）；编译速度慢（大项目、泛型膨胀明显）；生态成熟度不均（GUI、游戏、企业 CRUD、科学计算弱于老牌语言）；async 体验割裂（无内置运行时、Pin 等概念复杂）；人才池小、招聘成本高；写原型时容易与借用检查器「搏斗」。
5. **## ⚠️ 常见误区**（误区/正解格式，3 条）：
   - 「Rust 全面吊打其他语言」→ 是场景选择，不是全面替代。
   - 「学了 Rust 就不用学别的语言」→ 互操作（FFI、Wasm、PyO3）是常态。
   - 「Rust 写出来的程序一定最快」→ 性能上限同级，瓶颈常在算法与架构。
6. **## 📝 一句话总结**：Rust 用编译期的严格换来运行期的安全与性能，在系统、后端、基础设施、高性能组件场景优势明显，但要为学习曲线与编译时间付费。
