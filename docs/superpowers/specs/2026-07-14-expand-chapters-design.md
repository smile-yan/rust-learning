# 教程内容扩展设计方案

## 目标

将 `js/chapters.json` 中每个章节的理论内容扩展为原来的 2-3 倍，提升教程的详细程度、可读性和趣味性；同时为 Rust 的核心概念章节增加 SVG 示意图，让抽象的语法和内存模型更直观。

## 范围

- **所有 53 个章节**的 `theory` 字段都需要扩展文案。
- **核心概念章节**额外增加 1-3 张 SVG 示意图。
- 现有 `code` 和 `hint` 字段保持不变，仅可能在 `theory` 中对代码进行逐行讲解。
- 不改动前端渲染逻辑、`app.js` 或构建流程。

## 核心概念章节（配图范围）

1. 基础入门：函数、控制流、所有权、结构体、枚举与模式匹配、错误处理、集合、切片、迭代器、模块与包管理
2. 中等应用：泛型、Trait、生命周期、闭包与迭代器、智能指针、并发基础
3. 高级应用：unsafe Rust、异步编程、Web 开发基础、并发模式与设计
4. Q & A：所有权、生命周期、栈和堆、RAII 等涉及内存模型的问答章节

配图数量按内容复杂度决定，预计总共生成 40-60 张 SVG。

## SVG 图示风格

- 格式：SVG 1.1，文件形式存放在 `images/` 目录。
- 引用方式：在 `theory` 中使用标准 Markdown 语法，例如 `![所有权转移](images/ownership-move.svg)`。
- 视觉风格：浅色卡片背景 + 高对比色块 + 清晰文字，确保在深色/浅色主题下都能看清。
- 不使用外部字体或网络资源，所有图形和文字均为内联矢量元素。

## 内容结构

每个章节扩展后的 `theory` 建议按以下结构组织：

1. **引入**：用一个生活化的类比或场景引入主题。
2. **概念图解**：用 SVG 或文字描述展示核心结构。
3. **深入讲解**：分小节详细解释语法、规则和注意事项。
4. **常见误区**：列出初学者容易踩的坑。
5. **一句话总结 / 口诀**：帮助记忆。

同时会适度加入表情符号（✅ ⚠️ 💡 🦀 🔧 等）提升阅读体验。


## 实施方案

1. 按四个模块（基础入门、中等应用、高级应用、Q & A）拆分为独立子任务。
2. 使用并行子代理分别生成：
   - 扩展后的 `theory` Markdown 文本；
   - 核心概念 SVG 文件。
3. 合并结果到 `js/chapters.json`，保留原有 JSON 结构。
4. 使用 `jq` 或 Python 校验 JSON 语法。
5. 启动本地后端，在浏览器中检查渲染效果和图片加载情况。

## 质量验证

- JSON 无语法错误，可正常解析。
- 每个章节 `theory` 字数达到原来的 2 倍以上。
- SVG 图片路径正确，能被前端正常加载。
- 核心概念章节的示意图没有文字截断或重叠。
- 在本地 `http://localhost:9001` 预览无样式异常。

## 明确排除

- 不修改 `app.js`、不引入新的构建工具。
- 不替换现有代码示例（只做解释性补充）。
- 不使用外部 CDN 图片，所有图示本地化。
